# Bynry QA Automation Case Study Solution

**Repository for QA Automation Engineering Intern case study submission.**

## Overview

This repo demonstrates solutions for the three parts of the case study:
- **Part 1**: Debugging flaky Playwright tests for user login and multi-tenant access.
- **Part 2**: Test automation framework design for multi-platform, multi-tenant B2B SaaS platform.
- **Part 3**: API + UI integration test for project creation flow.

## Tools & Technologies Used

- **Test Runner**: pytest (flexible markers, fixtures, parallel execution)
- **UI Testing**: Playwright (Chrome, Firefox, Safari, mobile support, headless)
- **API Testing**: requests library (REST API calls, authentication)
- **Cross-browser Testing**: BrowserStack (for mobile web validation)
- **Configuration**: YAML files + environment variables
- **Reporting**: Allure (HTML test reports)
- **Page Object Model**: Modular page classes for maintainability
- **CI/CD Ready**: pytest-xdist for parallel execution
